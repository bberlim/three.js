How to use node material

NodeMaterial allows the creation of Shader Graph as well as procedural materials compiled for different outputs such as WebGL and WebGPU.

How it works?

Each NodeMaterial is formed by several Nodes, you can customize a Material using existing Nodes and also create your own.

All Nodes can be interconnected and can vary between one or more inputs, but always a single output. Nodes can contain values such as Numbers, Textures, Attributes of a Geometry, Mathematical Functions, among others, customized as you prefer.

Each Node is responsible for emitting an output that can vary depending on the input arguments or the output expected by the parent Node.

Every Node emits a snippet in the output, which is a small snippet of code that represents the result of the code. This code is generated from the Node.generate() function.

Node

It is the smallest fraction of a NodeMaterial, every NodeMaterial is made up of several Nodes, and every AnythingNode.js is extended from a Node.js

A Node can be updated every frame or simply by a constant in the code. Most Nodes are mathematical expressions.

ShaderNode

NodeMaterial offers its own abstraction language so you don't have to rewrite Shaders for different outputs. This also makes it easier to optimize the shaders themselves for the resources used in Three.js, since NodeBuilder can optimize resources for updates made in the library. Syntax is a fusion of JS + GLSL, you can use JS to create the variables and flow while using GLSL functions.

FunctionNode

For more specific purposes you can also create custom functions using the default language of your backend, ie GLSL for WebGL or WGSL for WebGPU.
